# WarpParse 0.22

Dear users,

We are pleased to announce the release of WarpParse 0.22. This update focuses on **runtime operability, remote project synchronization, KnowDB capability enhancements, and better observability and rule authoring workflows**.

## Highlights

- Completed the runtime admin surface with standardized status query and reload entry points.
- Further unified the remote project bootstrap, update, and hot-reload workflow.
- Expanded KnowDB, WPL, and monitoring documentation to make the recommended usage path clearer.

## Added

### Runtime Admin Surface

Added standardized operational entry points for online instances:

- `POST /admin/v1/reloads/model`
- `wproj engine reload`

[Runtime admin usage guide](../10-user/01-cli/operations/admin.md)

### Remote Sync and Hot Reload

The remote rule repository workflow is now documented as a more complete operational path:

1. `wproj init --repo <REPO> [--version <VERSION>]`
2. `wproj conf update`
3. `wproj engine reload`

[Remote project bootstrap and rule hot-reload SOP](../10-user/01-cli/operations/project-sync.md)

### Self Update

Added the CLI self-update workflow:

```bash
wproj self check
wproj self update --yes
```

[`wproj` project tool guide](../10-user/01-cli/cli/project.md)

### Remote Database Support

KnowDB now supports remote PostgreSQL / MySQL through `[provider]`:

- Supports either directory-based SQLite authority or a remote database
- Supports PostgreSQL / MySQL through `connection_uri`
- Uses `[cache]` as the unified result-cache configuration section

[KnowDB configuration guide](../10-user/02-config/07-knowdb_config.md)

### Observability and Pipeline Inspection

With VictoriaMetrics, VictoriaLogs, and Wp-Monitor, WarpParse now has a more complete inspection path for checking:

- Whether the pipeline is processing data correctly
- Whether MISS data is abnormal
- Whether downstream output is fluctuating

Related docs:

- [Wp-Monitor](../10-user/10-monitor/Wp-Monitor.md)

### WPL Updates

Added guidance for combining `json_like`, `json`, and `bad_json`, making class-of-JSON input handling clearer.

Related docs:

- [WPL language reference](../10-user/03-wpl/04-language-reference.md)

## Changed

### CLI Manual Structure

The product overview, CLI usage, runtime admin surface, and remote sync SOP have been consolidated into a more unified entry point.

Related docs:

- [CLI guide](../10-user/01-cli/README.md)

### Clearer Stats and Troubleshooting Paths

The workflow around `wproj data stat`, `wproj data validate`, and `wproj rescue stat` is now more complete for day-to-day troubleshooting.

Related docs:

- [`wproj` project tool guide](../10-user/01-cli/cli/project.md)
- [`wprescue` and rescue data guide](../10-user/01-cli/cli/rescue.md)
