# WarpParse 0.22

亲爱的用户们，

我们很高兴地宣布 WarpParse 0.22 版本发布！本次更新主要聚焦于 **运行时可运维性、远端工程同步、知识库能力补强，以及监控与规则体验增强**。

## 本次更新亮点

- 运行时管理面能力补齐，支持标准化状态查询与重载。
- 远端工程初始化、更新、热重载路径进一步统一。
- 知识库、WPL 与监控链路文档更加完整，正式使用路径更清晰。

## Added

### 运行时管理面

新增围绕在线实例的标准化运维入口：
- `POST /admin/v1/reloads/model`
- `wproj engine reload`

[运行时管理面使用说明](../10-user/01-cli/operations/admin.md)

### 远端工程同步与热更新 SOP

围绕远端规则仓库，当前已经形成较完整的运维路径：

1. `wproj init --repo <REPO> [--version <VERSION>]`
2. `wproj conf update`
3. `wproj engine reload`

[远程工程拉取与规则热更新 SOP](../10-user/01-cli/operations/project-sync.md)

### 自更新能力

新增 CLI 自更新使用路径：

```bash
wproj self check
wproj self update --yes
```
[`wproj` 项目工具使用指南](../10-user/01-cli/cli/project.md)

### 知识库支持远程数据库

知识库当前已支持通过 `[provider]` 接入远程 PostgreSQL / MySQL：

- 支持目录式 SQLite authority 与远程数据库二选一
- 支持通过 `connection_uri` 接入 PostgreSQL / MySQL
- `[cache]` 统一用于结果缓存配置

[KnowDB 配置说明](../10-user/02-config/07-knowdb_config.md)

### 监控与链路观察方式

配合 VictoriaMetrics、VictoriaLogs 与 Wp-Monitor，当前已经形成更完整的链路观察入口，可用于查看：

- 链路是否正常处理数据
- MISS 数据是否异常
- 下游输出是否波动

相关文档：

- [Wp-Monitor](../10-user/10-monitor/Wp-Monitor.md)

### WPL `json_like` 路由说明

补充了 `json_like`、`json`、`bad_json` 的配合说明，便于处理类 JSON 输入。

相关文档：

- [WPL 语言参考](../10-user/03-wpl/04-language-reference.md)
