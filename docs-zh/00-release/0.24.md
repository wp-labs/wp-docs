# WarpParse 0.24

亲爱的用户们，

我们很高兴地宣布 WarpParse 0.24 版本发布！本次更新主要聚焦于 **HTTP 接入能力、远端工程组织方式、错误治理体验，以及 OML/SQL 解析细节修复**。

## 本次更新亮点

- 新增 HTTP Source，补齐通过 HTTP 接口接收外部数据推送的接入路径。
- 远端工程支持双仓库模式，可分别同步 `models/` 与 `infra/`。
- 错误治理体系升级，CLI 报错更完整、更易读。
- OML `take(...)` 与 SQL 参数解析行为进一步修正，规则执行更稳定。

## Added

### HTTP Source

新增通用 `HTTP Source`，支持通过 HTTP 接口接收外部推送数据。

当前支持：

- `json` / `ndjson` 两种输入格式
- `Content-Type` 与 `fmt` 参数协同指定格式
- `gzip` 压缩请求体
- 基于 `port` + `path` 的接入配置

相关文档：

- [HTTP 源配置](../10-user/05-connectors/01-sources/05-http_source.md)

### 双仓库远端工程模式

远端工程同步能力新增双仓库模式，支持将：

- `models/`
- `infra/`

分别从两个独立的 Git 仓库同步。

这使得规则内容与基础设施配置可以分开维护，适合更正式的团队协作与发布流程。

相关文档：

- [远程工程拉取与规则热更新 SOP](../10-user/01-cli/operations/project-sync.md)

### Postgres Source 兼容性增强

补充 `Postgres Source` 的特性继承支持，并增强对低版本场景的兼容性适配。

这一项更新主要面向已有工程升级时的平滑过渡，减少因配置差异导致的接入调整成本。

## Changed

### 错误治理体系升级

本版本全面升级错误治理体系，`orion-error` 升级到 `0.8` 主线。

这带来的直接变化包括：

- 错误信息附带更完整的路径上下文
- CLI 报错信息更集中、更易读
- 日常排障时更容易定位具体配置或规则位置

## Fixed

### OML `take(...)` 行为修复

同步上游修复后，`take(...)` 在目标记录中已生成字段的消费行为更加正确，并修正了同名字段场景下的取值顺序。

### OML / SQL 参数解析增强

同步上游修复，增强以下 SQL 相关场景的解析稳定性：

- `group_concat(...)`
- `string_agg(...)`
- `IN (...)`
- `take(field)`
- `__temp_var`

### 运行时稳定性修复

修复了部分场景下的运行时稳定性问题。
